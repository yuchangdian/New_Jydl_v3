# 如何把 `data.cpp` 加入构建，并让 `compile_commands.json` 出现它

## 先说结论

`compile_commands.json` 不是手动改的。

你真正要改的是：

- `entry/src/main/cpp/CMakeLists.txt`

改完以后重新构建，CMake 会自动重新生成：

- `entry/.cxx/default/default/debug/arm64-v8a/compile_commands.json`

只要 `data.cpp` 被正式加入 `add_library(...)`，重新构建后，`compile_commands.json` 里就会自动出现 `data.cpp`。

## 你现在为什么没有 `data.cpp`

当前 [CMakeLists.txt](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/cpp/CMakeLists.txt) 里这一行是：

```cmake
add_library(entry SHARED napi_init.cpp tcp_client.c)
```

这里只有：

- `napi_init.cpp`
- `tcp_client.c`

没有 `data.cpp`，所以：

- 构建系统不会编译 `data.cpp`
- `compile_commands.json` 里也不会生成 `data.cpp` 的编译命令

这两个结果是完全一致的。

## 正确改法

把这一行改成：

```cmake
add_library(entry SHARED
    napi_init.cpp
    tcp_client.c
    data.cpp
)
```

## 为什么要这样改

`add_library(entry SHARED ...)` 后面的文件列表，就是这个动态库真正参与编译的源文件列表。

谁在这个列表里：

- 谁就会被编译
- 谁就会出现在 `compile_commands.json`

谁不在这个列表里：

- 谁就不会被编译
- 谁就不会出现在 `compile_commands.json`

## 改完后你要做什么

### 第 1 步：保存 `CMakeLists.txt`

文件位置：

- `entry/src/main/cpp/CMakeLists.txt`

### 第 2 步：重新构建工程

你要重新跑一次原生构建，目的是让 `.cxx` 目录重新生成编译数据库。

也就是说，**不是去编辑 `compile_commands.json`，而是重新生成它。**

### 第 3 步：检查 `compile_commands.json`

重新构建后，打开：

- `entry/.cxx/default/default/debug/arm64-v8a/compile_commands.json`

这时候应该能看到类似：

```json
{
  "file": "E:\\baidu\\arkTS_code\\New_Jydl_v3\\entry\\src\\main\\cpp\\data.cpp",
  ...
}
```

## 如果改了 `CMakeLists.txt` 还是没出现

说明构建系统还没刷新干净。

这时按下面顺序处理：

1. 再执行一次完整构建
2. 关闭并重新打开 DevEco 的工程索引
3. 必要时删除 `entry/.cxx` 后重新构建

第三步要谨慎，但它的作用很直接：

- 删掉旧的 CMake 缓存
- 让原生构建配置重新生成

## 你不要做的事

不要直接手改：

- `compile_commands.json`

原因很简单：

1. 它是构建产物，不是源码
2. 你手改完，下次构建还会被覆盖
3. 手改不能真正让 `data.cpp` 参与链接，只会制造假象

## 一句话理解

你要记住这一条：

**`compile_commands.json` 是结果，不是入口。**

入口是：

- `CMakeLists.txt`

你只需要把 `data.cpp` 加进 `add_library(...)`，然后重新构建，`compile_commands.json` 就会自动跟着变。
