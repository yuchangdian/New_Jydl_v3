# 宽频谐波触摸起滑卡顿单次续约修正说明

## 问题现象

- 宽频谐波页面在静止状态下每秒刷新一次数据。
- 滚动过程中已经基本不再卡顿。
- 但从“不滑动”到“开始滑动”的瞬间，仍然会偶发卡一下，像是手指按下时正好撞上了一次页面刷新。

## 本次处理

### 1. 把循环定时器改成单次续约

- 原来使用 `setInterval` 周期刷新。
- 现在改成 `setTimeout` 单次执行，回调结束后再续约下一次。
- 这样在 `Touch Down` 时调用 `stopHarmonicDisplayRefreshTimer()`，可以直接取消下一次刷新，不会继续保留固定周期的排队回调。

### 2. 增加刷新代次失效机制

- 新增 `harmonicDisplayRefreshGeneration`。
- 每次停止刷新时先自增代次。
- 定时回调执行前先比对代次，不一致就直接丢弃。
- 这样即使边界时刻有旧回调已经排队，也不会继续执行后续刷新链路。

### 3. 行节点 key 改回静态

- 之前行 key 绑定了实时数值，导致每次值变化都整行卸载再重建。
- 现在 `DataRow` 已经直接接收动态字符串参数，因此 row key 改回静态 `${sectionIndex}-${rowIndex}`。
- 这样刷新时只更新行内文本，不再因为 key 变化整行重建。

### 4. 去掉每秒页面赋值日志

- 删除了 `load assign` 这类每秒打印一次的页面侧调试日志。
- 避免日志输出本身放大 UI 线程抖动。

## 影响

- 刷新仍保持约 `1s` 一次。
- 手指按下后会尽快停止下一拍刷新。
- 重新松手后，页面会先补一次最新数据，再恢复后续定时刷新。
- 行数据更新仍然保留，不会因为静态 key 导致再次出现“值不刷新”。
