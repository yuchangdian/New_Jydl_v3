# 宽频谐波取值索引修正说明

## 问题

`broadband_harmonic.ets` 页面读取谐波电流、电压时，原来代码虽然用了 `componentIndex * 3`，但“每个分量占 3 个槽位”的语义没有直接写出来，后续继续改页面时很容易把数组理解错。

用户明确说明了 C 结构体的真实布局：

- `Ia[0]` = A 相工频频率
- `Ia[1]` = A 相工频幅值
- `Ia[2]` = A 相工频相位
- `Ia[3]` = A 相谐波 1 频率
- `Ia[4]` = A 相谐波 1 幅值
- `Ia[5]` = A 相谐波 1 相位
- 后面按 3 个一组继续排

`Ib[]`、`Ic[]`，以及电压侧 `Ua[]`、`Ub[]`、`Uc[]` 也是同样规则。

## 本次修改

在 `entry/src/main/ets/pages/broadband_harmonic.ets` 中：

- 增加了显式常量：
  - `HARMONIC_GROUP_VALUE_COUNT = 3`
  - `HARMONIC_FREQUENCY_OFFSET = 0`
  - `HARMONIC_AMPLITUDE_OFFSET = 1`
  - `HARMONIC_PHASE_OFFSET = 2`
- 增加 `getHarmonicComponentBaseIndex(componentIndex)`，明确规定：
  - `componentIndex = 0` 对应工频，起始下标是 `0`
  - `componentIndex = 1` 对应谐波 1，起始下标是 `3`
  - `componentIndex = 2` 对应谐波 2，起始下标是 `6`
- 电压、电流都统一走 `createMetricFromValues(...)`，不再在两个函数里分别写魔法数字。
- 数组长度校验也同步改成 `HARMONIC_COMPONENT_COUNT * HARMONIC_GROUP_VALUE_COUNT`，避免继续散落 `3` 这个硬编码。

## 当前页面取值规则

页面 section 与数组索引的关系现在是：

- 第 1 个 section“工频分量” -> `[0, 1, 2]`
- 第 2 个 section“谐波分量1” -> `[3, 4, 5]`
- 第 3 个 section“谐波分量2” -> `[6, 7, 8]`
- 依次类推

每个 section 三行分别对应：

- A 相 -> `Ia / Ua`
- B 相 -> `Ib / Ub`
- C 相 -> `Ic / Uc`

也就是：

- A 相“工频分量”的频率取 `Ia[0]`
- A 相“谐波分量1”的频率取 `Ia[3]`
- B 相“工频分量”的频率取 `Ib[0]`
- C 相“谐波分量1”的频率取 `Ic[3]`

## 结果

这次修改的重点不是改页面结构，而是把 C 结构体的索引规则直接固化到 ArkTS 代码里，避免后续再把“分量号”和“数组下标”混淆。
