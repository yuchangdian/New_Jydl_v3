# 保护定值整定页启动提速与区号 01-20 实现说明

## 本次改动目标

1. 保持保护定值整定页现有单页布局不变。
2. 把整定页区号规则从 `00-19` 改成 `01-20`。
3. 把整定页运行期数据模型从“全量 20 区 store”改成“当前区 profile + 区文件存储”。
4. 避免整定页启动时在模块加载阶段就把整张大表实例化。
5. 保存当前区时继续回写旧 `relay_setting_profiles_store_v2` 镜像，保证浏览页现阶段还能读到数据。

## 主要实现点

### 1. 区号规则

- `RelaySetting_SetSelect.ets` 已改为只允许 `01-20`。
- 整定页默认区号改为 `01`。
- 标题默认文案也改为 `—整定 01 区保护定值`。
- 在 `RelaySettingProfiles.ets` 中新增了整定页专用映射：
  - `relayNormalizeSetZoneCode`
  - `relaySetZoneCodeToIndex`
  - `relaySetZoneCodeForIndex`
  - `relaySetZoneCodeToLegacyZoneCode`

说明：

- 原有 `relayNormalizeZoneCode / relayZoneCodeToIndex / relayZoneCodeForIndex` 仍保留旧 `00-19` 语义，避免直接把浏览页现有逻辑打断。
- 整定页已切到新的 `01-20` 映射。

### 2. 启动提速

- 新增 `RelaySettingSetSchema.ets`。
- 原来在 `RelaySettingSet.ets` 顶层直接构造的 `RELAY_MAIN_SECTIONS / RELAY_MONITOR_SECTIONS` 已迁出。
- 现在改成：
  - 先 import schema 工厂
  - 首次进入页面时在 `relayEnsureSchemaReady()` 里惰性构造
  - 再做 `buildRelayFieldBindings()` 和 `relayBindFieldSpecs()`

这样可以避免路由刚进入页面时就因为模块顶层初始化大表而卡住。

### 3. 当前区文件存储

- 整定页新增目录：
  - `filesDir/relay_setting`
- 当前区文件命名：
  - `zone_01.json`
  - `zone_02.json`
  - ...
  - `zone_20.json`
- 迁移完成标记文件：
  - `migration_01_20.done`

整定页现在优先读取当前区文件，不再在打开页面时反序列化整包 20 区数据。

### 4. 旧数据迁移

- 如果当前区文件不存在，会回退到旧存储读取：
  - 先读 `relay_setting_profiles_store_v2`
  - 再回退读旧行文本 `relay_setting_set_store`
- 旧区号按区序号平移：
  - 旧 `00 -> 新 01`
  - 旧 `19 -> 新 20`

此外还加了后台迁移：

- 页面首批内容出来后，再异步把 20 个区补写成新区文件。
- 全部写完后写入 `migration_01_20.done`。

### 5. 旧镜像兼容

- 每次整定页保存当前区后，都会重新构建旧镜像 `RelaySettingProfilesStore`。
- 旧镜像仍按旧区号 `00-19` 写回 `relay_setting_profiles_store_v2`。
- 这样浏览页当前不改 UI 的情况下，仍能继续通过旧链路读到整定页最新保存的数据。

注意：

- 我在构建旧镜像时做了兼容处理。
- 如果某些区的新文件还没迁完，会先回退到旧存储取值，不会因为当前区先保存而把其它区覆盖成默认值。

### 6. 首屏渲染策略

- 页面进入后先显示 loading。
- 当前区数据就绪后先显示前 4 个 section。
- 剩余 section 通过 `setTimeout(0)` 分批追加渲染。
- 页面结构、顺序、字段布局都没改，只改了挂载时机。
