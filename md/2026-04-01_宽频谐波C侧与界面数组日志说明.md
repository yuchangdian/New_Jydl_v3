# 宽频谐波 C 侧与界面数组日志说明

## 本次目的

为了对比 `C` 侧收到的谐波数组，和 `broadband_harmonic.ets` 页面最终显示出来的电压、电流值，这次在两层都增加了按数组下标展开的日志。

日志格式统一为：

- `Ua[0]=...`
- `Ua[1]=...`
- `Ua[2]=...`
- 依次到 `Ua[29]`

电流同理：

- `Ia[0]=...`
- `Ia[1]=...`
- `Ia[2]=...`
- 依次到 `Ia[29]`

## C 侧日志

文件：

- `entry/src/main/cpp/tcp_client.cpp`

触发时机：

- 收到 `RemoteMetry_HarmonicU` 并更新 `HarmonicU_Dsip` 后
- 收到 `RemoteMetry_HarmonicI` 并更新 `HarmonicI_Dsip` 后

日志关键字：

- `C侧电压原始值`
- `C侧电流原始值`

输出内容：

- `Ua / Ub / Uc`
- `Ia / Ib / Ic`

每 6 个元素打一行，避免单条日志过长。

## 界面侧日志

文件：

- `entry/src/main/ets/pages/broadband_harmonic.ets`

触发时机：

- 页面拿到 `getHarmonicVoltageDisplayData()` 返回值并赋值后
- 页面拿到 `getHarmonicCurrentDisplayData()` 返回值并赋值后

日志分两类：

1. 原始数组日志

- `界面拿到的原始电压值`
- `界面拿到的原始电流值`

这部分打印的是 ArkTS 从 native 拿到的原始数组值。

2. 界面显示值日志

- `界面显示电压值`
- `界面显示电流值`

这部分打印的是已经赋给页面 `sections` 后，界面实际使用的：

- 频率
- 幅值
- 相位

打印时仍按 `Ua[0] ~ Ua[29]`、`Ia[0] ~ Ia[29]` 这种顺序展开，便于和 C 侧逐项对比。

## 说明

当前日志按真实数组下标打印，也就是从 `0` 开始，不是从 `1` 开始。这样可以直接对应 C 结构体里的实际元素位置，例如：

- `Ua[0]` 是工频频率
- `Ua[1]` 是工频幅值
- `Ua[2]` 是工频相位
- `Ua[3]` 是谐波 1 频率

如果后面调试时需要改成从 `1` 开始的显示标签，再单独调整日志标签即可。
