## 目的

这份文档用于快速定位两个问题：

1. 程序从哪里开始
2. TCP 是在哪里被调用的

---

## 一、程序启动入口

### 1. 模块入口先看这里

文件：

- `entry/src/main/module.json5`

关键位置：

- `entry/src/main/module.json5:6`
- `entry/src/main/module.json5:17`
- `entry/src/main/module.json5:20`
- `entry/src/main/module.json5:21`

关键配置：

```json5
"mainElement": "EntryAbility",
"pages": "$profile:main_pages",
...
"name": "EntryAbility",
"srcEntry": "./ets/entryability/EntryAbility.ets"
```

这说明应用启动后，先进入 `EntryAbility`，不是直接从某个 `pages/*.ets` 页面开始。

### 2. 首页页面列表看这里

文件：

- `entry/src/main/resources/base/profile/main_pages.json`

关键位置：

- `entry/src/main/resources/base/profile/main_pages.json:3`

内容里第一页是：

```json5
"pages/Index"
```

这表示 `Index.ets` 是首页页面资源之一，但它仍然是由 `EntryAbility` 去加载，不是系统直接跳进去。

### 3. 真正把首页加载出来的是这里

文件：

- `entry/src/main/ets/entryability/EntryAbility.ets`

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:160`
- `entry/src/main/ets/entryability/EntryAbility.ets:161`
- `entry/src/main/ets/entryability/EntryAbility.ets:166`

关键代码：

```ts
onWindowStageCreate(windowStage: window.WindowStage): void {
  windowStage.loadContent('pages/Index', (err) => {
    ...
  });
}
```

所以程序启动链路可以按下面理解：

`module.json5 -> EntryAbility -> onWindowStageCreate() -> loadContent('pages/Index') -> Index.ets`

---

## 二、首页页面本身做了什么

文件：

- `entry/src/main/ets/pages/Index.ets`

说明：

- `Index.ets` 是首页 UI
- 文件顶部有 `@Entry`
- 这个页面主要负责界面布局和显示
- 当前没有在这个页面里直接调用 TCP

也就是说：

TCP 不是在 `Index.ets` 里启动的，而是在 `EntryAbility` 的生命周期里启动的。

## 三、TCP 在 ArkTS 层是在哪里调用的

文件：

- `entry/src/main/ets/entryability/EntryAbility.ets`

### 1. 先导入 native 模块

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:4`

代码：

```ts
import * as tcpClientModule from 'libentry.so';
```

这一步表示 ArkTS 侧要调用 native 导出的 `libentry.so`。

### 2. 定义了 TCP 相关接口

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:12`

代码里定义了：

- `startTcpClient(host, port)`
- `stopTcpClient()`
- `isTcpClientRunning()`

### 3. 统一加载 TCP 模块的方法

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:50`

方法：

```ts
private loadTcpClientModule(): TcpClientModule | undefined
```

这个方法会按顺序尝试：

1. 直接使用 `import * as tcpClientModule from 'libentry.so'`
2. 如果失败，再尝试 `requireNapi('entry', true, 'com.example.new_jydl_v3/entry')`
3. 再尝试 `requireNapi('entry', true)`
4. 最后尝试 `requireNapi('entry')`

### 4. 真正启动 TCP 的调用点

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:123`
- `entry/src/main/ets/entryability/EntryAbility.ets:134`

关键代码：

```ts
private startNativeTcpClient(): void {
  const nativeModule = this.loadTcpClientModule();
  ...
  const started = nativeModule.startTcpClient(TCP_SERVER_HOST, TCP_SERVER_PORT);
}
```

其中目标地址写死在文件顶部：

- `entry/src/main/ets/entryability/EntryAbility.ets:9`
- `entry/src/main/ets/entryability/EntryAbility.ets:10`

```ts
const TCP_SERVER_HOST = '127.0.0.1';
const TCP_SERVER_PORT = 9000;
```

### 5. 程序在什么时候启动 TCP

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:167`

代码：

```ts
this.startNativeTcpClient();
```

它出现在：

```ts
windowStage.loadContent('pages/Index', ...)
```

的成功回调里。

也就是说：

首页加载成功后，才会启动 TCP 客户端。

### 6. 程序在什么时候停止 TCP

关键位置：

- `entry/src/main/ets/entryability/EntryAbility.ets:156`
- `entry/src/main/ets/entryability/EntryAbility.ets:157`
- `entry/src/main/ets/entryability/EntryAbility.ets:180`
- `entry/src/main/ets/entryability/EntryAbility.ets:181`

代码：

```ts
onDestroy(): void {
  this.stopNativeTcpClient();
}

onWindowStageDestroy(): void {
  this.stopNativeTcpClient();
}
```

这表示应用销毁或窗口销毁时会停止 TCP。

## 五、TCP 真正实现在哪里

文件：

- `entry/src/main/cpp/tcp_client.c`

这是 TCP 客户端的核心实现文件。

### 1. 启动入口

关键位置：

- `entry/src/main/cpp/tcp_client.c:156`

函数：

```c
int tcp_client_start(const char *host, int port)
```

这个函数会：

1. 保存服务器地址和端口
2. 创建 `pthread` 线程
3. 线程入口是 `tcp_client_thread_main`

### 2. 后台线程主循环

关键位置：

- `entry/src/main/cpp/tcp_client.c:83`

函数：

```c
static void *tcp_client_thread_main(void *arg)
```

这个线程里做的事情是：

1. 读取全局保存的 `host` 和 `port`
2. 调用 `tcp_client_connect(host, port)` 建立连接
3. 连接成功后循环 `recv(...)`
4. 收到数据后打印日志
5. 断开后如果还允许运行，就等待 3 秒再次重连

### 3. 建立连接的位置

关键位置：

- `entry/src/main/cpp/tcp_client.c:44`

函数：

```c
static int tcp_client_connect(const char *host, int port)
```

这里使用：

- `getaddrinfo`
- `socket`
- `connect`

来建立 TCP 连接。

### 4. 接收数据的位置

关键位置：

- `entry/src/main/cpp/tcp_client.c:117`

关键代码逻辑：

```c
ssize_t received_size = recv(socket_fd, recv_buffer, sizeof(recv_buffer) - 1, 0);
```

收到数据后会打印：

```c
TCP_LOGI("received: %{public}s", recv_buffer);
```

### 5. 停止 TCP 的位置

关键位置：

- `entry/src/main/cpp/tcp_client.c:186`

函数：

```c
void tcp_client_stop(void)
```

这个函数会：

1. 把 `g_should_run` 改成 `false`
2. 关闭 socket
3. 等待线程退出

## 六、`libentry.so` 是怎么来的

文件：

- `entry/src/main/cpp/CMakeLists.txt`

关键内容：

```cmake
add_library(entry SHARED napi_init.c tcp_client.c)
```

这表示：

- `libentry.so` 是由 `napi_init.c` 和 `tcp_client.c` 编译出来的
- ArkTS 侧导入的 `libentry.so`，本质上就是这两个 C 文件编译后的 native 模块

## 七、最短查看路径

如果你只想快速看懂“从启动到 TCP”的整条链，按下面顺序看文件就够了：

1. `entry/src/main/module.json5`
2. `entry/src/main/ets/entryability/EntryAbility.ets`
3. `entry/src/main/ets/pages/Index.ets`
4. `entry/src/main/cpp/napi_init.c`
5. `entry/src/main/cpp/tcp_client.c`

## 八、一句话总结

这套代码不是在首页 `Index.ets` 里调用 TCP，而是：

`应用启动 -> EntryAbility 创建窗口 -> 加载 pages/Index -> loadContent 成功 -> EntryAbility 调用 startNativeTcpClient() -> 进入 libentry.so -> napi_init.c -> tcp_client.c`
