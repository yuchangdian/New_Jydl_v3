# data.cpp 去掉 Qt 改成标准 C++ 的操作指导

## 目标

这份文档的目标只有一个：

**把 `entry/src/main/cpp/data.cpp` 及其关联声明里的 Qt 依赖去掉，改成标准 C++ 写法，同时保持原有逻辑功能不变。**

这里说的“逻辑功能不变”具体指：

1. 全局数据仍然表示同一批业务状态
2. SOE 相关函数仍然根据同样的 `code` 返回同样的文本含义
3. 结构体、宏、CRC、业务枚举含义不变
4. 只是把 Qt 类型和 Qt 字符串格式化方式换成标准 C++ 实现

---

## 当前现状

当前 `data.cpp` 不是“完全没改”，而是已经处于一种 **半替换、不可编译** 的状态：

1. `QString` 已经被替换成了 `std::string`
2. `QQueue` 已经被替换成了 `std::queue`
3. 但大量字符串格式化代码仍然保留着 Qt 风格：
   - `.arg(...)`
   - `QString::number(...)`
4. `data.h` 和 `IEC104.h` 里的声明还没同步改

这会导致一个核心问题：

**你现在的 `data.cpp` 既不是有效的 Qt 版本，也不是有效的标准 C++ 版本。**

所以正确顺序不是继续“零碎替换”，而是：

1. 先统一返回类型
2. 再统一队列类型
3. 再统一字符串格式化实现
4. 最后同步头文件声明

## 一、先明确哪些 Qt 依赖要被替换

### 1. `QString`

当前涉及位置：

- `data.cpp` 里的 `SOE_StartString` / `SOE_ActString` / `SOE_PhaseString` / `SOE_3ValString` / `SOE_2ValString` / `SOE_1ValString`
- `IEC104.h` 里的这些函数声明
- `setting.h` 里原本的 `Password`

标准 C++ 替换目标：

- `QString` -> `std::string`

### 2. `QQueue`

当前涉及位置：

- `data.cpp` 全局队列定义
- `data.h` 队列 extern 声明

标准 C++ 替换目标建议：

- `QQueue<T>` -> `std::deque<T>`

为什么这里建议 `std::deque`，不是直接坚持 `std::queue`：

1. `QQueue` 本质上更接近“可前后操作的序列容器”
2. `std::queue` 是适配器，功能更少，不能遍历
3. 如果后面你还要保留“队列长度、访问头尾、可能调试查看内容”的空间，`std::deque` 更稳

如果你已经确认这些队列以后只会做严格 FIFO 的 `push/pop/front`，也可以用 `std::queue`。  
但为了“尽量不改变后续使用方式的扩展空间”，我建议统一改成 `std::deque<T>`。

### 3. `QString::number(...)`

当前 `data.cpp` 里大量使用了这种模式，但已经被错误替换成了：

```cpp
string::number(...)
```

这是不存在的。

标准 C++ 替换目标：

- 使用 `std::ostringstream`
- 固定小数位时配合 `std::fixed` 和 `std::setprecision`

### 4. `.arg(...)`

这是 `QString` 独有接口，`std::string` 没有。

标准 C++ 替换目标：

- 自己写一个小的格式化帮助函数
- 或者直接用 `std::ostringstream` 拼接

为了减少出错，我建议你写一个最小帮助函数，不要在每个 `case` 里手写重复格式化逻辑。

## 四、最关键：把 Qt 字符串格式化替换成标准 C++

这是这次真正最容易改坏的地方。

## 第 7 步：先增加两个帮助函数

我建议你在 `data.cpp` 顶部、全局变量定义之后，先加下面两个帮助函数。

### 1. 固定小数格式函数

```cpp
static std::string ToFixedString(float value, int precision = 2)
{
    std::ostringstream oss;
    oss << std::fixed << std::setprecision(precision) << value;
    return oss.str();
}
```

### 2. 占位符替换函数

```cpp
static void ReplaceFirst(std::string &text, const std::string &from, const std::string &to)
{
    std::size_t pos = text.find(from);
    if (pos != std::string::npos) {
        text.replace(pos, from.length(), to);
    }
}
```

### 3. 三个常用模板格式化函数

因为你当前 `data.cpp` 里主要是 `%1`、`%2`、`%3` 三种场景，建议你继续加：

```cpp
static std::string Format1(const char *pattern, float v1)
{
    std::string text(pattern);
    ReplaceFirst(text, "%1", ToFixedString(v1));
    return text;
}

static std::string Format2(const char *pattern, float v1, float v2)
{
    std::string text(pattern);
    ReplaceFirst(text, "%1", ToFixedString(v1));
    ReplaceFirst(text, "%2", ToFixedString(v2));
    return text;
}

static std::string Format3(const char *pattern, float v1, float v2, float v3)
{
    std::string text(pattern);
    ReplaceFirst(text, "%1", ToFixedString(v1));
    ReplaceFirst(text, "%2", ToFixedString(v2));
    ReplaceFirst(text, "%3", ToFixedString(v3));
    return text;
}
```

### 为什么这么做

因为你原来的 Qt 逻辑是：

```cpp
QString("A=%1, B=%2").arg(...).arg(...)
```

这里的关键不是 Qt 本身，而是：

1. 保留原模板文本
2. 按顺序替换 `%1` `%2` `%3`
3. 数值仍然保留两位小数

上面这几个帮助函数正好是在不改变原表达语义的前提下，把 Qt 换成标准 C++。

## 第 8 步：按三类情况替换 `data.cpp`

## 情况 A：纯文本返回

例如：



```cpp

```

标准 C++ 改成：

```cpp
content = Format1("A相电压动作值：%1V", tNode.Value1);
```

## 情况 C：两个或三个数值占位符

原 Qt 写法类似：

```cpp
content = QString("零序电流电压动作值：%1A,%2V")
  .arg(QString::number(tNode.Value1, 'f', 2))
  .arg(QString::number(tNode.Value2, 'f', 2));
```

```cpp
content = Format2("零序电流电压动作值：%1A,%2V", tNode.Value1, tNode.Value2);
```

```cpp
content = Format3("ABC三相电压动作值：%1V，%2V，%3V",
                  tNode.Value1, tNode.Value2, tNode.Value3);
```

## 六、这几个地方不要误改

## 1. 不要改 switch 的 case 值

例如：

- `Cd_CurrentStartMoment`
- `Cd_Act_Uabc_RMS`
- `Cd_Act_IoUo_RMS`

这些都是业务编码，不是 Qt 内容。

### 原则

只改：

1. 返回类型
2. 字符串类型
3. 格式化方式

不要改 case 常量和分支结构。

---

## 2. 不要改中文文本本身的语义

比如：

- `"电流启动元件动作"`
- `"ABC三相电压动作值：%1V，%2V，%3V"`

这些文本就是业务输出的一部分。  
去 Qt 化不等于要改文案。

### 原则

保留原来每个分支对应的中文文本，仅替换格式化实现。

---

## 3. 不要把所有地方都改成 `std::to_string`

`std::to_string(1.23f)` 默认格式不稳定，经常会出现很多无用小数位。

你原来的 Qt 语义是：

- `'f'`
- `2`

也就是固定两位小数。

所以要用：

- `std::ostringstream`
- `std::fixed`
- `std::setprecision(2)`

而不是直接 `std::to_string`。

---

## 4. 不要继续保留这种半替换写法

下面这些都必须从文件里清掉：

1. `string::number(...)`
2. `std::string(...).arg(...)`
3. `queue<...>` 但头文件里还是 `QQueue<...>`
4. `data.cpp` 返回 `std::string`，`IEC104.h` 还声明 `QString`

这种状态会让文件越来越乱，最后你很难确认到底哪一层出错。

## 九、最容易漏掉的坑

## 坑 1：编码显示看起来像乱码

你现在命令行里看到的有些中文像乱码，不代表源码本身一定错了，很多时候只是终端编码问题。

### 正确做法

判断是否“逻辑被改坏”，不要靠终端显示判断，应该靠：

1. 代码里的原始文本是否被误改
2. 函数分支是否还对应原来的 case

也就是说：

**不要因为终端显示乱码，就顺手把所有中文字符串重写一遍。**

这会引入不必要的风险。

---

## 坑 2：把 `std::queue` 当成 `QQueue` 完全等价

如果你后面确实要继续操作这些队列内容，`std::queue` 没有迭代器，调试和访问灵活度很低。

所以这里我更建议你统一换成：

```cpp
std::deque<T>
```

这样后面如果需要：

1. `push_back`
2. `pop_front`
3. `front`
4. `back`
5. `size`

都更自然。

---

## 坑 3：只改 `data.cpp`，不改声明

这是最容易发生的错误。

你如果只把：

- `data.cpp` 改成 `std::string`

但：

- `IEC104.h` 还是 `QString`
- `data.h` 还是 `QQueue`

那编译一定会继续失败。

所以必须按“声明先统一、实现后落地”的顺序做。

这次你真正要做的，不是“把 `QString` 机械替换成 `std::string`”，而是：

**把 Qt 的字符串表示方式、队列类型和格式化机制，完整迁移成标准 C++ 的等价实现。**

只有这样，才能既去掉 Qt，又不改掉原有逻辑。
