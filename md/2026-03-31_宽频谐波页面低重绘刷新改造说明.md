# 宽频谐波页面低重绘刷新改造说明

## 问题原因

`broadband_harmonic.ets` 之前把整套 `displaySections` 作为页面实时状态：

- 每轮刷新都会重新创建整棵 section/row 对象树
- 外层 `ForEach` 的 key 又绑定了整段显示文本
- 任意一个值变化，整段 section 都会被销毁再重建

页面本身还是一个长表格，这种刷新方式在持续滚动和频繁重绘时就容易发卡。

## 本次改造

这次把页面改成“静态结构 + 扁平字符串状态”：

- 表格结构继续使用固定的 `BROADBAND_HARMONIC_SECTIONS`
- 实时显示值拆成 8 组 `@State string[]`
- 外层 section key 改成稳定的索引 key
- 内层 row key 保持稳定的 `sectionIndex-rowIndex`

这样页面结构不再跟数据值绑定，只更新单元格文本。

## 刷新逻辑调整

`loadNativeHarmonicDisplayData()` 现在不再构造整套 section 数据，而是：

1. 从 native 读取谐波电压和谐波电流
2. 直接映射成 8 组字符串数组
3. 生成 `lastHarmonicDisplaySignature`
4. 只有签名变化时才回填页面状态

这样可以避免：

- 设备值没变时重复赋值
- 无意义的列表重绘

## 定时器调整

页面刷新频率从 `500ms` 调整为 `1000ms`：

- 页面进入时仍然先读一次缓存
- 然后主动发一次请求
- 后续定时执行 `load -> request`

这次只降页面刷新频率，不改 native 接口。
