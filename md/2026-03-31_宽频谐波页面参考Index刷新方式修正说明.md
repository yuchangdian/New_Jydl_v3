# 宽频谐波页面参考 Index 刷新方式修正说明

## 背景

`broadband_harmonic.ets` 前面几版一直在 `sections: HarmonicSectionData[]` 这层做页面刷新。

虽然 native 日志已经证明：

- 请求每 `0.5` 秒发出
- 对端每 `0.5` 秒回包
- `HarmonicU_Dsip / HarmonicI_Dsip` 持续更新

但页面仍然存在：

- 一版能刷但卡
- 一版不卡但不刷

用户提示参考 `Index.ets` 的实现方式。

## Index 页的稳定做法

`Index.ets` 的刷新思路很简单：

1. native 读取最新值
2. 格式化成简单字符串
3. 逐个赋值给 `@State string`
4. UI 直接绑定这些字符串

也就是：

- 不把复杂嵌套对象直接作为页面渲染状态核心
- 而是把显示值拆成简单文本状态

## 本次修正

`broadband_harmonic.ets` 改成和 `Index.ets` 同一思路：

### 1. 页面结构改成静态

保留 `HarmonicSectionData` 只作为静态表格结构定义，不再把它作为实时显示状态核心。

### 2. 实时显示值改成简单字符串数组状态

新增 8 组 `@State string[]`：

- 电压频率
- 电压幅值
- 电压相位
- 电压含有率
- 电流频率
- 电流幅值
- 电流相位
- 电流含有率

### 3. native 数据先映射成字符串，再回填页面

`loadNativeHarmonicDisplayData()` 现在的流程是：

1. 从 native 取 `HarmonicU_Dsip / HarmonicI_Dsip`
2. 先生成临时 `nextSections`
3. 再把 `nextSections` 扁平化成 8 组字符串数组
4. 最后赋值给页面 `@State`

### 4. UI 渲染只按索引读取字符串状态

表格行仍然按静态 section/row 生成，但每个单元格显示内容直接从：

- `voltageFrequencyTexts`
- `voltageAmplitudeTexts`
- `voltagePhaseTexts`
- `voltageRatioTexts`
- `currentFrequencyTexts`
- `currentAmplitudeTexts`
- `currentPhaseTexts`
- `currentRatioTexts`

里读取。

## 结果

这版的核心变化是：

- 页面结构静态
- 显示内容简单化
- 刷新逻辑和 `Index.ets` 同型

这样可以绕开 `sections` 这种嵌套对象列表在当前页面上的不稳定刷新问题。
