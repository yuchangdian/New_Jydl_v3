# 项目启动入口说明

## 先说结论

这个项目不是从 C/C++ 的 `main()` 开始的，而是一个 OpenHarmony ArkTS 应用。

它的启动链是：

1. 系统先读取 `module.json5`
2. 根据 `mainElement` 找到 `EntryAbility`
3. 进入 `EntryAbility.ets`
4. 先执行 `onCreate(...)`
5. 再执行 `onWindowStageCreate(...)`
6. 在这里调用 `loadContent('pages/Index')`
7. 页面真正进入 [Index.ets](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/ets/pages/Index.ets)
8. 如果 ArkTS 侧加载了 `libentry.so`，native 层还会走 `RegisterEntryModule -> Init`

## 入口链逐层说明

### 第 1 层：模块入口配置

文件：

- [module.json5](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/module.json5)

这里最关键的是：

- `mainElement: "EntryAbility"`
- `srcEntry: "./ets/entryability/EntryAbility.ets"`

这表示应用模块启动时，系统先创建：

- `EntryAbility`

## 第 2 层：应用 Ability 入口

文件：

- [EntryAbility.ets](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/ets/entryability/EntryAbility.ets)

这里要重点看两个函数。

### `onCreate(...)`

这是 Ability 创建时最先进入的生命周期函数。

它更偏向“应用初始化入口”，适合做：

- 模块检查
- 全局配置初始化
- 环境准备

### `onWindowStageCreate(...)`

这是窗口创建完成后的入口。

你这个项目里，真正把首页挂上去的是这里：

```ts
windowStage.loadContent('pages/Index', ...)
```

所以如果你从“界面从哪里开始显示”这个角度看，真正的界面启动入口就是：

- `onWindowStageCreate(...)`

## 第 3 层：首页页面入口

文件：

- [Index.ets](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/ets/pages/Index.ets)

这里有两个关键点：

### `@Entry`

它表示这个页面组件可以作为页面入口被加载。

### `build()`

`build()` 是页面 UI 树的总入口。

也就是说：

- 页面加载到窗口后
- ArkUI 会从 `build()` 开始构建整棵界面树

## 第 4 层：native 模块入口

文件：

- [napi_init.cpp](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/cpp/napi_init.cpp)

如果你还关心 `libentry.so` 是怎么接进来的，那 native 侧入口是：

### `RegisterEntryModule()`

这是 so 装载时的注册入口。

### `Init(...)`

这是 NAPI 导出入口，决定 ArkTS 最终能调用哪些 native 函数。

比如：

- `startTcpClient`
- `stopTcpClient`
- `isTcpClientRunning`

都是在这里挂到 `exports` 上的。

## 你可以把它理解成这样

### ArkTS 应用启动入口

- [EntryAbility.ets](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/ets/entryability/EntryAbility.ets) 的 `onCreate(...)`

### ArkTS 界面启动入口

- [EntryAbility.ets](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/ets/entryability/EntryAbility.ets) 的 `onWindowStageCreate(...)`

### 首页页面入口

- [Index.ets](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/ets/pages/Index.ets) 的 `@Entry + build()`

### native 模块入口

- [napi_init.cpp](/E:/baidu/arkTS_code/New_Jydl_v3/entry/src/main/cpp/napi_init.cpp) 的 `RegisterEntryModule()` 和 `Init(...)`
