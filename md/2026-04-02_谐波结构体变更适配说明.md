# 谐波结构体变更适配说明

## 变更背景

`data.h` 里的两个结构体已经从旧格式：

- `Ua[30] / Ub[30] / Uc[30]`
- `Ia[30] / Ib[30] / Ic[30]`

改成了新格式：

- `Ua_Frequency[10], Ua_RMS[10], Ua_Angle[10]`
- `Ub_Frequency[10], Ub_RMS[10], Ub_Angle[10]`
- `Uc_Frequency[10], Uc_RMS[10], Uc_Angle[10]`
- `Ia_Frequency[10], Ia_RMS[10], Ia_Angle[10]`
- `Ib_Frequency[10], Ib_RMS[10], Ib_Angle[10]`
- `Ic_Frequency[10], Ic_RMS[10], Ic_Angle[10]`

## 本次适配策略

为了不把 `broadband_harmonic.ets` 整页重写一遍，这次采用兼容适配方式：

1. C++ 层读取新的 `Frequency / RMS / Angle` 三组数组
2. 按旧页面使用的顺序重新拼接成 30 位扁平序列
3. 再通过 NAPI 导出给 ArkTS 页面

重新拼接后的顺序保持不变：

- `[0]` 工频频率
- `[1]` 工频幅值
- `[2]` 工频相位
- `[3]` 谐波1频率
- `[4]` 谐波1幅值
- `[5]` 谐波1相位
- 依次类推

## 本次修改文件

- `entry/src/main/cpp/telemetry_napi.cpp`
  - 新增把 `Frequency / RMS / Angle` 交错拼接为 30 位数组的逻辑
  - 页面拿到的 `Ua / Ub / Uc / Ia / Ib / Ic` 仍然保持旧接口不变

- `entry/src/main/cpp/tcp_client.cpp`
  - 把旧的 `value.Ua[0]` 之类访问，改成从新结构体字段读取
  - C 侧日志也改为按交错后的顺序输出，方便继续和页面日志逐项比对

## 结果

结构体已经适配到新定义，但 ArkTS 页面层继续沿用原先的 30 位数组读取逻辑，所以页面频率、幅值、相位、含有率的现有计算代码不需要跟着大改。
