# 宽频谐波页面卡顿优化说明

## 问题

上一版为了确保 `broadband_harmonic.ets` 每 `0.5` 秒能看到新值，给 `ForEach` 的 key 加了 `harmonicRenderVersion`。

这样虽然能刷新，但副作用很明显：

- 每次刷新都会把整页谐波列表整块销毁再重建
- 页面滚动和重绘会明显发卡

## 本次优化

### 1. 去掉整页强制重建

移除了：

- `@State harmonicRenderVersion`
- `ForEach` key 中的版本号

不再每 `0.5` 秒强制重建整页 UI。

### 2. 改成固定列表结构 + 动态读取当前状态

`SectionBlock` 和 `DataRow` 不再依赖临时传入的整块对象，而是通过：

- `getSectionByIndex(sectionIndex)`
- `getRowByIndex(sectionIndex, rowIndex)`

按索引从当前 `@State sections` 里取最新值。

这样列表结构保持稳定，只让文本内容更新。

### 3. 增加重复数据过滤

`loadNativeHarmonicDisplayData()` 里增加了当前页面数据签名：

- `lastHarmonicDisplaySignature`

如果本轮读到的数据和上一轮完全一样，就直接跳过，不再重复赋值和重绘。

## 结果

当前页面刷新链路是：

1. 每 `0.5` 秒读取 native 最新缓存
2. 只有当数据实际变化时才更新 `sections`
3. 列表结构保持稳定，不再整页重建

这样可以保留 `0.5` 秒刷新，同时显著降低页面卡顿。
