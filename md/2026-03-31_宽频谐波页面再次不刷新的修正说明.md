# 宽频谐波页面再次不刷新的修正说明

## 问题

上一轮为了减轻卡顿，我把 `broadband_harmonic.ets` 的行渲染改成了：

- `SectionBlock(sectionIndex)`
- `DataRow(sectionIndex, rowIndex)`
- 再通过 `getSectionByIndex()` / `getRowByIndex()` 去取当前值

这样虽然减少了整页重建，但现场反馈成了“页面又不刷新”。

## 原因

这类写法在当前 ArkUI 页面里不够稳。

页面虽然有新的 `@State sections`，但行内文本不再直接依赖 `ForEach` 当前传入的对象，而是改成了额外的方法间接取值。实际效果是：

- 列表结构是稳定的
- 但文本刷新没有稳定传递到每个单元格

## 本次修正

把页面回到更稳的方式：

1. `ForEach(this.sections, (section, index) => ...)`
2. `SectionBlock(section, sectionIndex)`
3. `ForEach(section.rows, (row, rowIndex) => ...)`
4. `DataRow(row)`

也就是：

- 仍然保留稳定 key
- 仍然保留重复数据过滤
- 但每一行显示内容重新直接绑定 `row` 对象

这样页面更新路径更直接，ArkUI 对文本更新更稳定。

## 保留的优化

这次没有回退的内容：

- 每 `0.5` 秒刷新一次
- 先读缓存，再发下一轮请求
- `lastHarmonicDisplaySignature` 重复数据过滤

## 结果

当前这版目标是：

- 恢复稳定刷新
- 同时避免上一版整页强制重建带来的明显卡顿
